What each role can do

DEVELOPER

  • → All issues (open, closed, all states)
  • → Add / remove labels
  • → Comment as [Developer]
  • → Record decisions with rationale
  • → Invite designers via one-time links
  • → Monitor + revoke designer sessions

DESIGNER

  • → Only designer-input labeled issues
  • → Comment as [Designer]
  • → Record design decisions
  • → View collaboration info
  • → No GitHub account needed
  • → Server proxies all GitHub calls

MCP Tools

Tool Parameters Role Description
list_issues state? both List issues. Designer sees only designer-input issues.
get_issue issue_number both Issue details + comments with role prefixes parsed.
add_comment issue_number, body both Post a comment prefixed with [Developer] or [Designer].
record_decision issue_number, decision, rationale? both Record a structured decision block on an issue.
get_collaboration_info — both Active sessions, pending invites, fresh invite URL.
label_issue issue_number, label, action developer Add or remove a label from an issue.
