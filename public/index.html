DEVELOPER

  • → Full issue access (all states)
  • → Add / remove labels
  • → Post comments as [Developer]
  • → Record decisions with rationale
  • → Invite designers via one-time links
  • → Monitor active sessions

DESIGNER

  • → Sees only designer-input issues
  • → Post comments as [Designer]
  • → Record design decisions
  • → View collaboration state
  • → Auth via GitHub OAuth (invite link)

CLAUDE MCP CONFIG

After connecting, your setup page shows the exact snippet. It looks like this:

{
  "mcpServers": {
    "github-collab": {
      "url": "https://github-issue-collab-server.vercel.app/mcp",
      "headers": {
        "Authorization": "Bearer <api_key>"
      }
    }
  }
}

Designers get their own session token via the invite flow (get_collaboration_info tool).

MCP TOOLS

Tool Parameters Available to Description
list_issues state? both List issues. Designer sees only designer-input labeled issues.
get_issue issue_number both Get issue details + comments with role prefixes parsed.
add_comment issue_number, body both Post a comment prefixed with [Developer] or [Designer].
record_decision issue_number, decision, rationale? both Record a structured decision block on an issue.
get_collaboration_info — both Active sessions, pending invites, fresh invite URL for designers.
label_issue issue_number, label, action developer Add or remove a label from an issue.

ARCHITECTURE

Hosted (Vercel) — apps/server

  • GET /connect → GitHub App install redirect
  • GET /connect/callback → issue API key, setup page
  • POST /mcp → stateless JSON-RPC handler
  • GET /invite → designer OAuth redirect
  • GET /invite/callback → create designer session
  • POST /token → installation token broker

Local — packages/server

  • POST /mcp/developer — developer MCP (SDK transport)
  • POST /mcp/designer — designer MCP (SDK transport)
  • GET /auth — designer OAuth flow
  • GET /health — health check

Postgres (Vercel)

users · designer_sessions · invite_codes

github-issue-collab GET /health → { status: "ok" }
