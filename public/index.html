github-issue-collab

v0.0.1 / local MCP

Developer + Designer Collaboration

GitHub Issues.
Through Claude.
Together.

A local MCP server that lets developers and designers interact with GitHub issues directly inside Claude. Developers get full access. Designers see only what's relevant.

DEVELOPER

  • → Full issue access (all states)
  • → Add / remove labels
  • → Post comments as [Developer]
  • → Record decisions with rationale
  • → Invite designers via one-time links
  • → Monitor active sessions

MCP endpoint

POST /mcp/developer
Authorization: Bearer $DEV_SECRET

DESIGNER

  • → Sees only designer-input issues
  • → Post comments as [Designer]
  • → Record design decisions
  • → View collaboration state
  • → Auth via GitHub OAuth

MCP endpoint

POST /mcp/designer
Authorization: Bearer <session-token>

SETUP

01

GitHub App

Create a GitHub App with Issues (read/write) permission. Set the OAuth callback URL.

/auth/callback URL:
http://localhost:3000/auth/callback
02

Configure .env

GITHUB_APP_ID=...
GITHUB_APP_INSTALLATION_ID=...
GITHUB_REPO=owner/repo
GITHUB_APP_PRIVATE_KEY_PATH=./key.pem
GITHUB_APP_CLIENT_ID=...
GITHUB_APP_CLIENT_SECRET=...
DEV_SECRET=random-secret
PORT=3000
03

Start & Connect

# Start the server
npm run dev

# Generate designer invite
tsx apps/cli/src/index.ts invite

# Check sessions
tsx apps/cli/src/index.ts status

CLAUDE MCP CONFIG

Add to claude_desktop_config.json or your Claude Code MCP settings:

{
  "mcpServers": {
    "github-collab-dev": {
      "url": "http://localhost:3000/mcp/developer",
      "headers": {
        "Authorization": "Bearer <DEV_SECRET>"
      }
    }
  }
}

Designers connect to /mcp/designer with their session token obtained via the invite flow.

MCP TOOLS

Tool Parameters Available to Description
list_issues state? both List issues. Designer sees only designer-input labeled issues.
get_issue issue_number both Get issue details + comments with role prefixes parsed.
add_comment issue_number, body both Post a comment prefixed with [Developer] or [Designer].
record_decision issue_number, decision, rationale? both Record a structured decision block on an issue.
get_collaboration_info — both Active sessions, pending invites, server URLs.
label_issue issue_number, label, action developer Add or remove a label from an issue.

ARCHITECTURE

packages/server

  • index.ts — Express app + MCP wiring
  • tools.ts — 6 MCP tools (role-scoped)
  • auth.ts — OAuth flow + middleware
  • db.ts — SQLite sessions + invites
  • github.ts — GitHub API + token cache

apps/cli

  • serve — start MCP server
  • invite — generate designer invite
  • status — show active sessions
  • setup — print setup instructions

apps/server (Vercel)

Token broker — deployed separately, not part of local server.
